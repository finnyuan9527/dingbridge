# Security Policy

## 中文

DingBridge 处理企业身份、token、session 和钉钉应用密钥。请不要在公开 issue、PR、日志或截图中暴露：

- `.env`
- RSA 私钥
- 钉钉 AppSecret
- OIDC ClientSecret
- Refresh token、access token、ID token
- 真实用户手机号、邮箱、部门和内部域名

如果你发现安全问题，请通过私下渠道联系项目维护者，不要公开披露细节。报告中请包含影响范围、复现步骤和建议修复方式。

## English

DingBridge handles enterprise identity data, tokens, sessions, and DingTalk app secrets. Do not disclose the following in public issues, pull requests, logs, or screenshots:

- `.env`
- RSA private keys
- DingTalk AppSecret
- OIDC ClientSecret
- Refresh tokens, access tokens, or ID tokens
- Real user phone numbers, emails, departments, or internal domains

If you find a security issue, contact the maintainers privately instead of disclosing details publicly. Please include impact, reproduction steps, and suggested remediation.
